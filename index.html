Media queries Big brother of CSS 2.1 media types

Media queries

Allows testing for attributes
  • Viewport width/height
  • Device screen width/height
  • Device orientation
  • Aspect ratio
  • Color
  • Grid
  • Etc.
Example 

        <!-- In HTML link element -->
        <link rel="stylesheet" media="(max-width: 800px)" 
                  href="example.css" />

        /* In a stylesheet */
        img.jumbotron { display: block; }

        /* Hide large image on small devices */
        @media only screen and (max-width: 320px; ) {
          img.jumbotron { display: none; }
        }

Using preprocessing is often a good idea

A SASS mixin can be used to write it like this

/* Using a mixin which defines responsive rules */
.profile-pic {
  float: left;
  width: 250px;
  @include respond-to(handhelds) { width: 100% ;}
  @include respond-to(medium-screens) { width: 125px; }
  @include respond-to(wide-screens) { float: none; }
}

Modernizr

Modernizr can check for features and perform both presentational and functional polyfills 

Modernizr.load([{
    // Presentational polyfills
    test : Modernizr.fontface && Modernizr.canvas && Modernizr.cssgradients,
    nope : ['presentational-polyfill.js', 'presentational.css']
  },{
    // Functional polyfills
    test : Modernizr.websockets && window.JSON,
    // socket-io.js and json2.js
    nope : 'functional-polyfills.js',
    both : [ 'app.js', 'extra.js' ],
    complete : function () {
      // Run this after everything in this group has downloaded
      // and executed, as well everything in all previous groups
      myApp.init();
    }
  },
  // Run your analytics after you've already kicked off all the rest
  // of your app.
  'post-analytics.js'
]);
With Modernizr we can write queries like this: 

    /* Only hide the caption on non-touch devices */
    .no-touch .photo .caption { display: none; }
    .no-touch .photo:hover .caption { display: block; }

Tools of the trade
  • You can use Twitter Bootstrap to get a kick start.
  • Use Modernizr, or some alternative, for feature detection.
  • CSS preprocessors (SASS or LESS) can be helpful when writing media queries.
  • Device.js can help you if going adaptive.
  • The Responsinator to display your page on multiple devices. Screenfly is another alternative.
  • Chrome dev tools allows you to override user agent and device metrics
